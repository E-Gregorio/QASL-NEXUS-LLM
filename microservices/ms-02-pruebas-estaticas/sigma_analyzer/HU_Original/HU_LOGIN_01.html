Épica EP-001: Autenticacion y Control de Acceso
Enlace a documentacion de la Epica
Prioridad Alta
Estado Refinada
Descripción
Como usuario registrado del sistema
Quiero iniciar sesion con mis credenciales (correo electronico y contrasena)
Para acceder a las funcionalidades del sistema segun mi rol asignado
Usuarios / Roles Usuario registrado, Administrador, Operador
Reglas de Negocio BR1: El correo electronico debe tener formato valido (usuario@dominio.com)
BR2: La contrasena debe tener minimo 8 caracteres, incluyendo mayuscula, minuscula y numero
BR3: Despues de 3 intentos fallidos consecutivos, la cuenta se bloquea por 15 minutos
BR4: La sesion expira automaticamente despues de 30 minutos de inactividad
BR5: El sistema debe registrar cada intento de login (exitoso o fallido) en el log de auditoria
CU-001 | Caso de uso: Autenticacion de usuario
Precondiciones • El usuario debe estar registrado previamente en el sistema
• El servicio de autenticacion (Keycloak/OAuth) debe estar operativo
• La base de datos de usuarios debe estar disponible
Dependencias • Integracion con Keycloak (Access Manager)
• API de autenticacion del backend operativa
• Servicio de correo para recuperacion de contrasena

Escenarios de Prueba (Criterios de Aceptación)
E1: Login exitoso con credenciales validas
DADO que el usuario tiene una cuenta activa registrada en el sistema
CUANDO ingresa su correo electronico y contrasena correctos y presiona el boton Iniciar Sesion
ENTONCES el sistema autentica al usuario y lo redirige al dashboard principal segun su rol
E2: Login fallido por contrasena incorrecta
DADO que el usuario tiene una cuenta activa en el sistema
CUANDO ingresa su correo electronico correcto pero una contrasena incorrecta
ENTONCES el sistema muestra el mensaje de error "Credenciales invalidas" y no permite el acceso
E3: Bloqueo de cuenta por intentos fallidos
DADO que el usuario ha fallado 2 intentos de login consecutivos
CUANDO falla el tercer intento de login
ENTONCES el sistema bloquea la cuenta por 15 minutos y muestra el mensaje "Cuenta bloqueada temporalmente"
Dentro del Alcance
• Validacion de formato de correo electronico
• Validacion de contrasena segun politica de seguridad
• Bloqueo temporal de cuenta por intentos fallidos
• Redireccion al dashboard segun rol del usuario
Fuera del Alcance
